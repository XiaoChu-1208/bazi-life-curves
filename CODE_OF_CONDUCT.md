# Code of Conduct · 行为准则

> 本项目把 600 年前的命理学搬进 2026 年。我们认真对待"现代化"的承诺。

## 我们的承诺

我们致力于让 `bazi-life-curves` 的开发者社区与用户社区 **不分**：

- 性别 / 性别认同（cis / trans / non-binary / 不公开）
- 性取向（hetero / homo / bi / pan / ace / 不公开）
- 婚恋形态（已婚 / 未婚 / 不婚 / 一夫一妻 / 多元关系）
- 生育选择（已育 / 未育 / 不育）
- 年龄 / 残障 / 民族 / 国籍 / 体型 / 外貌
- 宗教信仰（含无宗教 / 无神论 / 不可知论）
- 受教育程度 / 经济水平 / 社会阶层
- 工作经验 / 技术水平

均能获得开放、欢迎、安全的参与体验。

## 不可接受的行为

- ❌ 在 issue / PR / discussion / 解读模板中使用"克夫 / 旺夫 / 妨夫 / 妨妻 / 命硬"等带价值判断的命理措辞
- ❌ 把"未婚 / 不婚 / LGBTQ+ / 不育"描述为命局缺陷
- ❌ 任何形式的人身攻击 / 骚扰 / 歧视性言论
- ❌ 公开他人隐私（真实姓名 / 八字 / 出生信息 / 经历）
- ❌ 性别 / 取向 / 婚恋的"劝告"（比如"你这个命应该早结婚"）

## 命理学的特殊约束

由于本项目处理的是"命理"这个高度敏感、容易触发宿命论 / 性别歧视的领域，我们**额外**要求：

1. **解读语言去价值化**：emotion 高 ≠ 婚姻顺利、emotion 低 ≠ 单身差，所有解读必须中性
2. **结构识别 ≠ 价值判断**：可以说"配偶星弱 → 关系密度低"，**禁止**说"配偶星弱 → 单身 = 差"
3. **可证伪点优先**：禁止"反正以后你会知道"式的不可证伪论断
4. **不预设对方性别**：emotion 解读必须强制声明"命局只反映关系结构和能量模式，不预设对方性别 / 是否结婚 / 是否生育"

## 执行

- 违反者首次：维护者私信提醒
- 二次违反：PR / issue / discussion 关闭
- 严重违反（公开人身攻击 / 暴露他人隐私）：直接 block 并报告 GitHub

## 报告

发现违反行为请在 issue 区开 `[CoC]` 前缀的 issue，或直接通过 GitHub 私信项目维护者。
