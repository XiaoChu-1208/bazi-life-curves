# 准确性保障协议（Accuracy Protocol）

本 skill 不靠 LLM「凭感觉」打分，准确性靠以下机制保障：

## 1. 三派独立 + 加权融合

每年的每个维度评分由三派独立计算：

- **扶抑派**：日主强弱 × 用神到位度
- **调候派**：寒暖燥湿 × 流年五行
- **格局派**：原局格局成败 × 大运是否破/成

融合规则（极差为三派给分中最大值减最小值）：

| 极差 | 置信度 | 最终值 | 备注 |
|---|---|---|---|
| ≤ 10 | high | 加权平均（默认等权） | 三派一致 |
| 11–20 | mid | 中位数 | 一派偏离 |
| > 20 | **low** | 中位数 × 0.85 + 50 × 0.15 | 偏离剧烈，向基线收敛 |

## 2. 三层独立打分（避免循环污染）

- **L0 原局基线**：人一生不变，纯由四柱算
- **L1 大运修正**：当前 10 年大运对原局的影响
- **L2 流年修正**：当年干支与原局 + 大运的互动

三层各自独立，禁止反向回写：流年得分不能调整大运得分，大运得分不能调整原局基线。

## 3. 历史回测（核心质量门）

`calibration/dataset.yaml` 内置 5–10 个**公开**历史人物八字 + 已发生关键节点：

- 算法每次改动 → 跑 `python scripts/calibrate.py`
- 命中率（方向正确）必须 ≥ `calibration/thresholds.yaml` 指定的阈值（默认 0.70）
- 量级误差 ≤ 1 档（高 / 中 / 低三档）的样本占比 ≥ 0.60
- 任一指标不达标 → `calibrate.py` 非零退出，CI 友好

## 4. 可追溯性

每年评分输出包含 `breakdown` 字段：

```json
{
  "year": 2027,
  "spirit_yearly": 68,
  "breakdown": [
    {"rule": "scoring_rubric.md#A.1", "delta": +12, "reason": "大运正印"},
    {"rule": "scoring_rubric.md#A.5", "delta": +10, "reason": "流年补足通关用神"},
    {"rule": "scoring_rubric.md#D.1", "delta": -5, "reason": "用神大运讫年"}
  ]
}
```

每条规则引用 `scoring_rubric.md` 章节号，用户质疑时可逐项追溯。

## 5. 已知边界

本 skill **不**做以下事：

- 不算流月 / 流日（颗粒度仅到流年）
- 不做合婚 / 比对（只算单人命盘）
- 不做严格占卜（这是经验性可视化，结果应作为「概率分布」而非「确定预测」）
- 不处理「真太阳时校正」过细的边界情况（接受 ±15 分钟误差，不影响柱位的输入即可）

## 6. 失败模式与降级

| 失败模式 | 检测 | 降级行为 |
|---|---|---|
| 三派分歧 > 30 | `divergence_high` 标记 | 该年置信度 = low，图上灰阶 |
| 大运换运过渡（首尾两年） | `transition` 标记 | 权重 × 0.7 |
| 入库 / 多重互动同发 | `interaction_dense` 标记 | 该年置信度降一档 |
| `solve_bazi.py` 检测到不合规输入（带身份字段） | `ValueError` | 拒绝执行，提示「请去掉身份信息」 |
