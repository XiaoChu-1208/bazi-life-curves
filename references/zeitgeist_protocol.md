# 时代叙事协议（Zeitgeist Protocol）

> 在命理事件的解读中，把单年事件**镶嵌进 5-10 年的时代区间**，用社会语境给命理结构提供"显化通道"。本协议规定 LLM 在 Step 3a 处理 `key_years` / `dayun_review` 时，如何引入时代叙事而**不破坏命局打分的客观性**。

## 1. 为什么需要这个协议

### 1.1 单年事件的认识论缺陷

现有 `multi_dim_xiangshu_protocol.md` 的 key_years 模板是按"单年 → 六维取象"组织的。这有两个问题：

- **过度精确的错觉**：命理事件（如"伤官见官"）的应期通常是 ±1 年的窗口，写成"1996 年发生 X"显得算命碰巧蒙对
- **缺乏社会维度**：相同的"伤官见官"，1996 年 vs 2025 年，最可能显化的具体形态完全不同——但当前模板没有让 LLM 利用这种信息

### 1.2 区间叙事的优势

| 维度 | 单年模式 | 区间模式 |
|---|---|---|
| 认知错觉 | 易显得"算命碰巧蒙对" | 显示"你处在一个真实的时代周期里" |
| 可证伪性 | 单点对错（噪声大） | 多点互相印证（聚合可信度） |
| 用户体感 | "啊那一年是这样" | "原来我那段日子是被这个浪带的" |
| 命理诚实度 | 假装精确到年 | 承认事件在区间里酝酿、触发、消化 |

## 2. 三个核心原则

### 2.1 「前事细 / 后事粗」（最重要）

**对过去的解读**：可以用时代背景做贝叶斯锐化——结合命局 + 时代 + 阶级 prior，给出**具体形态**（如"父母岗位变动"），允许"首选 + 备选"多个方向。

**对未来的预测**：只能给**方向 + 大类 + 避坑**，绝不可以推具体事件——未来的时代场景本身就有不可预测性，叠加命理事件后过拟合风险极高。

```
✓ 允许（过去）："1996 年伤官见官，最可能显化为父母单位下岗变动"
✓ 允许（未来）："2031 年财富次高峰，时代方向上对应 AI 应用层 / 银发经济"
✗ 禁止（未来）："2031 年你会通过某个 AI 创业项目赚到第一桶金"
```

### 2.2 「思维材料 ≠ 输出语言」

时代背景 + 阶级 prior 是 LLM 内部的**推理材料**，不是允许出现在用户可见文本中的**命名范畴**。

详见 `class_inference_ethics.md`。

### 2.3 「区间为骨，节点为肉」

每个 era_window 是一个完整叙事单元，命理节点（具体年份的事件）镶嵌在区间叙事内部，不再单独成段。

## 3. 区间叙事的结构

### 3.1 标准结构

每个 era_window 的解读必须按以下结构写：

```markdown
## {era_window.label} · 你 {age_at_start}–{age_at_end} 岁 · {era_window.span}

### 时代底色
{1 句 headline + 3-4 句背景描述（基于 era_window.keywords + LLM 训练知识展开）}

这段时间里有几个标志性的细节，你大概率经历过：
- {民俗志条目 1：物件 / 流行 / 热潮，按 folkways_protocol §3-4 推，标 confidence}
- {民俗志条目 2 ...}
- {5-8 条，优先 confidence=high，其次 mid，禁止 low}

### 命局给的几个具体节点

#### {年份 1}（{年龄}岁，{干支}，{大运}）· {命理事件名}
**命理读出**：{命局结构层面的解读，沿用 mangpai_events 等}
**放在大背景里看**：{结合 era_window.sub_phase + folkways + 阶级 prior 的具体化}
- **首选形态**：{最可能的具体显化（confidence=high/mid）}
- **备选形态 1**：{第二可能}
- **备选形态 2**：{第三可能}

#### {年份 2}（...）
... 

### 这段区间的累计影响（≥ 150 字）
{从命局结构 + 该区间所有命理节点 + 该区间集体情绪出发，
 推论命主在这段时间里"内化了什么底层叙事"}

### ⚠ 整段证伪点
如果你回忆这一整段（{era_window.span}）和上面描述的时代底色 / 命理节点完全对不上，
请告诉我——多半是命局的阶级 prior 或时辰判断有偏差，需要重新校准。
```

### 3.2 区间内部的命理节点选择

每个区间内挑 2-4 个命理节点（不要多于 4 个，否则稀释）：

- 优先选 `is_disputed=false`、`confidence=high`、`mangpai_event` 触发、或 `deviation` 大的年份
- 必须覆盖至少 2 个不同维度（不要 4 个节点都谈财富）
- 跨度应均匀分布在 era_window.span 上（不要 4 个节点都挤在头 2 年）

### 3.3 「累计影响」段是新增重点

这是区间叙事相对单年模式的**核心增量**——不只是讲"那一年发生什么"，而是讲"那段日子塑造了你内在什么底层结构"。

示例（来自 PLAN_zeitgeist_folkways.md §10.1）：

> 你在一个家庭和社会都在重新洗牌的时间窗里完成了童年到青春期的转换。命局的"印星护身"在这段时间起了关键作用——同样这段时代里，没有印星护身的孩子很多人留下了长久的不安全感（"我家就是从那时候开始走下坡路"），而你大概率内化的是"世界会变但我能稳住"的底层叙事。这种底层叙事会在你 30+ 岁面对类似不确定性时被反复唤起。

注意几点：
- 把命局结构（"印星护身"）和时代背景（"洗牌时期"）联立
- 给出"对比群体"（"没有印星护身的孩子"）来凸显命主的特异性
- 指向**当下/未来的延续**（"30+ 岁面对类似不确定性时被反复唤起"）——这是把"前事"和"后事"轻量打通的方式（不是预测，是指认底层模式）

## 4. 大运 × era_window 对齐策略

命局自带 10 年大运分段，时代区间是 5-10 年的社会分段，两者**通常会错位**。三种对齐情况，处理方式不同：

### 情况 A：大运 ≈ era_window（误差 < 2 年）

→ **共振区**，叙事最强烈，body 写最长（≥ 600 字）。这段是用户人生的"时代命运共振点"，应作为 dayun_review 的高光段。

```markdown
**辛巳大运（1995-2004） · 你 11-20 岁**

这段大运正好和「国企改制 + 入世前夜」时代区间重合——
你的成长节奏和整个社会的节奏完全同步...
```

### 情况 B：大运跨越两个 era_window

→ 大运分两段写，每段嵌入对应 era_window 背景。在两段交界处加一段"时代切换感受"。

```markdown
**辛巳大运（2000-2009） · 你 16-25 岁**

这段大运横跨两个时代区间：

前 3 年（2000-2002）你还在「入世前夜」的尾声里——...
后 7 年（2003-2009）切换到「入世红利 + 城市化加速」——...

时代切换感受：你大概率在 2002-2003 之交感受到一种"换天"——
{大运 + 时代切换的双重影响}
```

### 情况 C：一个 era_window 跨越两个大运

→ era_window 的时代背景写一段大的（在第一个大运的 dayun_review 里），下面分两段大运分别讲命理节点。

## 5. 区间叙事的颗粒度选择

### 5.1 era_windows_skeleton.yaml 的颗粒度

参考 `references/era_windows_skeleton.yaml`，**采用 7-10 年一段**（与大运量级对齐）。

不允许：
- ❌ 5 年一段（区间数量翻倍，叙事单元太短）
- ❌ 自由划分（与大运对齐困难）

允许：
- ✓ LLM 在某些特殊情况下（如战争、革命等剧烈断点）自行细分 sub_phases，但 era_window 的主 span 必须按骨架走

### 5.2 sub_phases 由 LLM 现场推

`era_windows_skeleton.yaml` 不写 sub_phases（区间内的子节奏），由 LLM 根据 keywords + 训练知识现场推：

```
era_window cn_reform_late_90s [1995, 2002] 内部，LLM 应推出 sub_phases：
- 1995-1997: 紧张 / 不确定（下岗潮主峰、香港回归、亚洲金融危机前夜）
- 1998-2000: 触底转折（98 洪水 / 朱镕基整顿 / 房改 / 互联网泡沫）
- 2001-2002: 入世前后，新的可能性开始浮现
```

这样保证：基础数据轻，推理灵活。

## 6. 非中国 / 年长用户的处理

### 6.1 非中国用户

`era_windows_skeleton.yaml` 含 4-5 个全球大事件主线（global_post_cold_war / global_post_911 / global_polarization / global_ai_age）。

LLM 处理非中国用户时：
1. 先匹配全球主线 era_window（提供宏观背景）
2. 再用训练知识**补充用户出生国的本土 era**（例如美国 80 年代 = Reaganomics + Cold War 末段；日本 80 年代 = 泡沫经济顶峰）
3. 标注 `confidence`：本土 era 一般是 mid（不如中国主线 high，因为骨架里没有锚定）
4. 在输出末尾加 caveat：「以上是按你出生地推的本土 folkways。如果你其实是移民家庭/在海外华人社区长大，请告诉我，会切换 prior。」

### 6.2 年长用户（出生在 era_windows_skeleton 起点之前）

`era_windows_skeleton` 中国主线起于 1977（cn_reform_dawn），全球主线起于 1990。

如果用户出生在 1950s 或更早：
1. LLM 用训练知识补充该 era（如 cn_cultural_revolution / cn_great_leap_forward）
2. 标 `confidence: mid`（民俗志细节不如近代精确）
3. 必须诚实说明"我对这一段的细节掌握程度有限，主要讲命理结构"

### 6.3 未来用户（出生在 2010 之后）

本 skill 不适用——命理本身建议 ≥ 6 岁才有可解读的过往轨迹。

## 7. 与其他协议的关系

| 协议 | 关系 |
|---|---|
| `multi_dim_xiangshu_protocol.md` | 本协议**替换**其 §3.1 的「单年六维取象模板」为「区间叙事模板」 |
| `folkways_protocol.md` | 本协议**调用**其规则来填充区间叙事中的"标志性细节" |
| `class_inference_ethics.md` | 本协议**遵守**其"思维材料 ≠ 输出语言"约束 |
| `dayun_review_template.md` | 本协议为大运评价提供"时代镶嵌"段 |
| `mangpai_protocol.md` | 命理节点的应事文本仍按盲派援引方式 |
| `prediction_protocol.md` | 本协议加强"前事细 / 后事粗"边界 |

## 8. 自检清单

写完一个 era_window 区间叙事后，按这个清单过一遍：

- [ ] 区间叙事按 §3.1 标准结构写完整（时代底色 + 标志性细节 + 命理节点 + 累计影响 + 整段证伪点）
- [ ] 标志性细节每条都标了 confidence；只有 high/mid 出现在输出
- [ ] 命理节点 2-4 个，跨度均匀，覆盖 ≥ 2 个维度
- [ ] 大运对齐情况判对（A/B/C）
- [ ] "累计影响"段 ≥ 150 字，把命局 + 时代联立讲
- [ ] "整段证伪点"段写了
- [ ] 没有出现身份命名（"你出身工人阶级"等）
- [ ] 未来年份没有写具体事件（保持"后事粗"）
- [ ] 非中国用户加了 caveat
