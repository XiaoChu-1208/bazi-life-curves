# 多维取象协议（Multi-Dimensional Xiangshu Protocol）

> "取象"是命理对干支组合做"画面化解读"的传统手法。本协议规定 LLM 在 Step 3a 给关键 10 年 / 整图综合分析 / 派别争议年份 写**多维取象**时必须遵守的框架与文案规范。

## 1. 为什么需要这个协议

LLM 在写命理解读时常见的两种错误：
- **空泛**：只说"这年事业上有变动"——用户不知道是好是坏、要做什么
- **臆造**：自由发挥"这年你会遇到一个穿白衣服的人"——超出脚本提供的事实

本协议解决两件事：
1. **强制覆盖六维**——避免漏掉某些维度的影响
2. **强制引用盲派应事**——避免离开机械事实的"幻觉自由发挥"

> **v6 更新 + v7 现代化**：曲线维度从 3 维（精神 / 财富 / 名声）扩展为 4 维（**+ 关系能量**）。
> 关系能量维度由 v9 adaptive_elicit D2「关系结构」维度题（偏好类型 + 对方反应模式 · 取代旧 v6/v7 R0 反询问，已退役）做命局取向校准 + 由 score_curves 单独打分通道生成 emotion_yearly / emotion_cumulative。
> 见 §4.2 / §4.6 / 文末「关系能量维度专项解读模板（§10）」。
>
> **v7 现代化要点**（必读）：
> 1. 维度名 emotion 字段保留（向后兼容），但**用户可见的措辞统一改为"关系能量"** —— 强调中性、不预设婚姻
> 2. 命局**只**反映关系结构 / 能量模式 / 偏好的互动模式 / 关系密度
> 3. 命局**不**反映对方的生理性别 / 是否结婚 / 几段关系 / 是否生育 / 关系是否被祝福
> 4. emotion 高 ≠ 婚姻顺利 / 关系幸福；emotion 低 ≠ 单身差 / 关系失败
> 5. 支持 `--orientation` 参数：hetero (默认) / homo / bi / none / poly，详见 fairness_protocol.md §9
> 6. 完整禁用措辞 + 现代化解读铁律见 fairness_protocol.md §10

## 2. 六维取象框架

每条取象解读必须明确覆盖以下 6 个维度（缺哪个明确说"该年度对该维度无显著事件"）：

| 维度 | 简称 | 关注什么 |
|---|---|---|
| **职业 / 事业** | career | 工作变动、晋升 / 降级、跳槽 / 创业、与上级 / 下属关系、行业切换 |
| **关系** | relationship | 配偶 / 伴侣、父母 / 子女、合伙人、密友、新关系出现 / 旧关系结束 |
| **财务** | finance | 现金流、负债 / 资产、合伙财、投资得失、不动产、被分财 |
| **健康 / 身体** | health | 慢性 → 急性切换、需要手术 / 体检异常、情绪躯体化、运动 / 饮食 / 睡眠节律 |
| **心理 / 状态** | psychology | 主导情绪、注意力分配、决策风格、自我认同、与"过去自己"的关系 |
| **时空 / 环境** | spacetime | 居所 / 城市变动、出差 / 移居 / 出国、空间风格切换、办公环境 |

> "心理"和"职业"经常合并写，但**必须各占一段**——它们的因果方向不同（心理是"内"，职业是"外"）。

## 3. 文案规范

### 3.1 强制结构

每个关键年份的多维取象**必须**按下面顺序写：

```markdown
**XXXX 年（X 岁，干支 / 大运）**

> 三派融合分：精神 X.X / 财富 X.X / 名声 X.X｜置信度 high/mid/low｜偏离基线 ±X
> 盲派烈度修正：spirit ±X, wealth ±X, fame ±X（若启用）
> 盲派应事：
>   - 【烈度】组合名（出处）：应事文本
>   - 【烈度】组合名（出处）：应事文本

【职业】... （≤ 3 句，必须落到具体行动 / 抉择）
【关系】... （≤ 3 句，要分清主动 / 被动）
【财务】... （≤ 3 句，要写"哪部分财在动"——固定资产 / 现金流 / 合伙财 / 期权 / 副业）
【健康】... （≤ 2 句，但若有 mangpai 健康相关事件则必写）
【心理】... （≤ 3 句，写主导情绪 + 决策风格变化）
【时空】... （≤ 2 句，写居所 / 城市 / 办公空间是否变）

**关键抉择 / 自处建议**（≤ 2 句，可操作的具体动作，不要心灵鸡汤）
```

### 3.1a 区间叙事模板（v7.5 新增 · key_years / dayun_review 推荐写法）

> 旧的 §3.1 单年六维模板仍保留供向后兼容，但**新的 key_years / dayun_review 推荐按本节的"区间叙事 + 节点镶嵌"模式**。原因详见 `zeitgeist_protocol.md` §1。

#### 何时用区间叙事 vs 单年模板

| 场景 | 推荐 |
|---|---|
| `dayun_review.body`（10 年大运段）| **必须**用区间叙事（按 `dayun_review_template.md`） |
| `key_years.body` 该年所在 era_window 上还有其它 key_year | **推荐**：把多个 key_year 合并放进所在 era_window 的区间叙事里 |
| `key_years.body` 该年是 era_window 内孤独 key_year | 可保留单年模板（§3.1），也可在 body 里嵌入"放在大背景里看"小节 |
| 未来年份（forecast_window 内）| **必须**用单年简化模板：`方向 + 大类 + 避坑`，禁止具体事件 |

#### 标准结构（按 era_window 写）

```markdown
## {era_window.label} · 你 {age_at_start}–{age_at_end} 岁 · {era.span}

### 时代底色
{1 句 headline + 3-4 句具体背景描述（不要抽象口号，要具体事件 + 氛围）}

这段时间里有几个标志性的细节，你大概率经历过：
- **{物件/事件 1}**（{时间窗}，{confidence: high/mid}）：{1 句具体描述}
- ...（5-8 条，按 folkways_protocol §2 跨 sub_layer 至少 3 层覆盖）

### 命局给的几个具体节点

#### {年份 1}（{年龄}岁，{干支}，{大运}）· {命理事件名}
**命理读出**：{命局结构层面解读，沿用 mangpai_events 等}
**放在大背景里看**：
- **首选形态**（high）：{把命理事件 + folkways + 阶级 prior 联立后的具体显化}
- **备选形态 1**（mid）：{第二可能}
- **备选形态 2**（mid）：{第三可能}

#### {年份 2}（...）
...

（每个 era_window 内 2-4 个节点，跨度均匀，覆盖 ≥ 2 个维度）

### 这段区间的累计影响
{≥ 150 字。把命局结构 + 该区间所有命理节点 + 该区间集体情绪联立。
 必须包含：
 ① "你内化了什么底层叙事"
 ② "对比群体"（"没有 X 命局结构的同代人多半是 Y，你是 Z"）
 ③ 指向当下/未来的轻量延续}

### ⚠ 整段证伪点
{说明"哪 1-2 个最 high confidence 的具体细节如果对不上，整段就该重写"}
```

#### 强制铁律

1. **前事细 / 后事粗**：本模板**只**用于过去年份。未来年份保留单年简化模板，且 only "方向+大类+避坑"
2. **思维材料 ≠ 输出语言**：阶级 prior 是内部 reasoning，**禁止**身份命名（详见 `class_inference_ethics.md`）
3. **三档置信**：每条 folkways 必须标 confidence；low 不出现在断言中
4. **五项自检**：每条 folkways 必须能回答 5 个问题（详见 `folkways_protocol.md` §4）
5. **跨 sub_layer 覆盖 ≥ 3 层**：标志性细节不能 5 条都同一类
6. **首选 + 备选**：每个命理节点至少 1 首选 + 1 备选，不止 1 个（避免过拟合）也不超过 3 个（避免稀释）
7. **累计影响段必写**：这是区间叙事相对单年模板的核心增量，必须 ≥ 150 字

#### few-shot 参考

写之前**强烈建议**先读这两份示例（模仿其颗粒度）：
- `references/folkways_examples/china_1995_2002.md`
- `references/folkways_examples/china_2010_2015.md`

### 3.2 引用盲派应事的方式

如果当年有 mangpai event，**必须**：

1. 在维度展开前先列出 mangpai 事件 block（用 `>` 引用块）
2. 至少有一条维度的展开**点名引用**该 mangpai 事件，例如：

> 【职业】今年你的"伤官见官"应期已经触发——具体表现是与直接上级 / 制度规则的剧烈摩擦。建议在 Q2 之前完成是否离职 / 是否切换业务线的决断，否则 Q3 流年再加压会被动收尾。

3. 引用时使用对应的语言：禁止把"伤官见官"改写成"你跟领导可能会有点不愉快"——保留命理术语 + 翻译应事。

### 3.3 禁止事项

❌ 不允许出现以下表达：
- "可能会"、"也许"、"建议关注一下"——空泛、非可证伪
- "你会遇到一个 X 岁的 X 性朋友"、"贵人会从北方来"——超出脚本事实
- "总体平稳，无特别大事"——若实际就是平年也要写"该年三派融合分接近基线，无 mangpai 事件触发，建议作为休整 / 学习 / 准备期"
- "命中注定 / 天意如此"——本 skill 立场是"命理是分析工具，不是宿命论"
- 把同一个 mangpai 事件在多个维度重复抄一遍——每个维度只抄相关角度

✅ 允许且鼓励：
- 引用具体年龄段（"40-43 岁这个区间"）
- 引用 dayun（"己亥大运的下半段"）
- 引用其他年份做对照（"和你 1996 年那次类似"）
- 给出可操作建议（"建议在该年 Q1 把 X 业务做收尾切割"）

## 4. 整图综合分析（图下方默认展开 section）

整图综合分析必须覆盖：

### 4.1 命局基本盘（≤ 4 句）
- 日主 + 强弱 + 用神 + 通关 + 月令格局
- **不允许**说"这是个好命 / 烂命"——只说结构特征

### 4.2 四条曲线的整体形状（v6 加感情维度）
- 精神曲线：高峰段 / 低谷段 / 中位区间，用具体年龄段说
- 财富曲线：同上
- 名声曲线：同上
- **感情曲线（v6 新增 · v9 adaptive_elicit D2 校准 · 旧 R0 反询问已退役）**：同上 + 必须援引 D2 关系结构题的命中情况
  · 若 D2 高置信（top1 ≥ 0.85）：感情维度的解读可以高置信度展开
  · 若 D2 中置信：在感情段开头注明"D2 部分对，以下感情解读偏一种取向"
  · 若 D2 未触发 / 低置信：感情段降权处理，写成"按命局原始读法可能是 X，但 D2 没对上 → 用户实际可能是 Y"，把双解都给出来
- 四条曲线的相互关系：是否同步？是否互补？是否对冲？特别要看**感情 vs 财富/名声**的对冲（典型场景：中年事业冲顶但感情低谷 / 早年感情顺但财富未起）

### 4.3 累积曲线的解读
- 累积线 vs 当年线 → 哪些年份的"积累"高于"当年体感"（"播种期"）
- 哪些年份"当年体感"高于"累积"（"收割期"）
- 一辈子整体的累积终点（age_end）落在哪个区间

### 4.4 三大转折点（用脚本给出的 turning points 选 top 3）
- 每个转折点写为什么是转折点（命理 + 盲派双重论据）
- 是上行转折还是下行转折
- 给出"前置准备"建议

### 4.5 命主定位（≤ 4 句）
基于命局结构 + 曲线形状，给出**身份定位**（不是吹捧也不是否定）：
- 适合的活动类型（用 yongshen 推导）
- 不适合的活动类型（用 jishen 推导）
- 长板 / 短板的代偿建议
- 一句话标签（例：「中后程发力的结构化产出者」「短期高光长期蓄势的连接者」）

## 5. 关键 10 年解读（每条 turning point 配一个 details）

每个 turning point 都要按 §3.1 的强制结构写一份。**禁止**只写 1-2 行带过。

如果该 10 年有多个 turning point，把它们合并写一份"X-X 岁段总览"，但每个 turning point 的具体年份必须分开列。

## 6. 派别争议年份的解读（在 dispute card 里默认收起）

按 `references/dispute_analysis_protocol.md` 的格式写。但要额外加一句：

- 该年是否有 mangpai event？如果有 → mangpai 视角下三派的分歧是什么意思？
- 例：扶抑派看跌（-30）、调候派看涨（+15）、格局派中性（0）；mangpai 触发"食神制杀"——则盲派站调候派一边（化煞为权 → 上行）。

## 7. 文案语气

- **学术化**但不晦涩——能用白话解释的术语就翻译，必要的术语保留并加 1 句解释
- **节制**——不渲染恐惧（"血光之灾"），也不渲染期待（"必发横财"）
- **直接**——给抉择就给抉择，不要"也许或许可能"
- **承认不确定**——置信度低 / 派别争议大的年份，明确说"该年三派分歧大，以下是综合推论"

## 8. 字数预算

- 单个关键年份的多维取象：400-700 字（六维 + 抉择建议）
- 整图综合分析：800-1200 字（5 个子节）
- 派别争议年份解读：250-400 字
- 不要超过这个上限——LLM 倾向写过长，反而稀释信号

## 10. 关系能量维度专项解读模板（v6 新增 · v7 现代化）

关系能量维度是 4 维中**最贴身**的一条，但也是用户最容易"自我美化 / 自我贬低"+ 古法最容易踩坑（克夫 / 旺夫 / 配偶预设）的一条，所以解读规则要严：

### 10.0 v7 现代化前置声明（必写）

「## 一生 · 关系能量」节的**第一段必写**以下声明（措辞可改但语义保留）：

> 命局只反映关系结构和能量模式，不预设对方性别、是否结婚、是否生育——这些是你的现代选择，不在命局之内。本工具的关系能量高低是中性描述：高 ≠ 婚姻顺利，低 ≠ 单身差。

### 10.1 「## 一生 · 关系能量」节的写作框架

```markdown
## 一生 · 关系能量（援引 R0：偏好类型 = X / 对方反应模式 = Y / 命中 = N/2 · orientation = Z / relationship_mode = W）

> [v7 现代化前置声明（见 10.0）]

**整体形态**：基线 X.X，relationship_mode = [outward_attractive / competitive_pursuit / nurture_oriented / ambiguous_dynamic / low_density / balanced / self_centered]，曲线波动主要由 [大运冲日支 / 大运合日支 / 大运为配偶星 / 大运比劫旺] 主导。
（用 score_curves 输出的 emotion_baseline + relationship_mode + 关键大运的 dy_p 信息）

**几个值得提的段：**
- **X-X 岁（大运 XX）**：关系能量走 [高/低]，原因是 [大运地支冲日支 / 大运为配偶星五行 / 等]
  → 这段对应你 R0 验证过的"对方反应模式 = Y"在该大运里的具体表现：[展开 1-2 句，去除任何"婚 / 配偶 / 异性缘"措辞]
- **X-X 岁（大运 XX）**：关系能量走 [高/低]，原因是 [...]
  → ...

**和事业 / 财富的关系**：
（必写：关系能量线和财富/名声/精神线在哪几年同步、哪几年对冲——这是用户真正关心的"长板换长板"问题）

**关键年份**：emotion 流年合冲日支的高峰 / 低谷（列 ≤ 3 个具体年份）
```

### 10.2 大运关系能量看点（每段大运的 1 行简评）

每段大运的解读里**必须**有"关系能量看点"一行（1 句话即可，不要长）：

> **大运 X 段（X-X 岁）关系能量看点**：[大运地支冲/合日支] + [大运为亲密关系核心星/比劫] → [简评]

**v7 示例**（去除了"夫星 / 妻星 / 克夫 / 异性缘"措辞）：

> **大运甲申（25-34 岁）关系能量看点**：申冲日支寅 + 甲为正官（亲密关系核心星透干 · hetero 取向下女命）→ 这 10 年关系结构「张力高 + 稳定性弱」，密度高但需要主动澄清边界

> **大运庚午（35-44 岁）关系能量看点**：午为日支三合 + 庚为食神（自我表达型）→ 这 10 年关系黏度高，但你会更主动定义关系形态而非被动接受

### 10.3 关键年份必带关系能量看点

任何 turning point（emotion_yearly 偏离 baseline ≥ 12 或 emotion sigma 异常大），**必须**在该年的多维取象中明确写：

> 【关系能量·v6】[当年地支合/冲日支] + [当年十神为亲密关系核心星/比劫] → 具体表现（**关系结构** 启动 / 沉淀 / 切换 / 重定义 等的命理预期，**不写**"结婚 / 离婚 / 生子"等社会决策类预测）

### 10.4 R0 命中级别 → 关系能量段权重

| R0 命中 | 关系能量段写法 |
|---|---|
| 2/2 | 高置信展开，可以下结论"这 10 年你大概率反复被 X 类型吸引 + 进入 Y 模式的关系结构" |
| 1/2 | 中置信，开头注明"R0 部分对，以下以 [对的那条] 为主" |
| 0/2 | 低置信，**必须双解**："按命局原始读法是 X，但 R0 没对上 → 也可能是 Y，建议用户自行印证（也可能是 orientation 取向跟默认不同）" |

### 10.5 关系能量段禁止事项（v7 大改 · 在 §3.3 基础上严格扩充）

❌ **绝对禁用措辞**（fairness_protocol.md §10.2 完整清单）：
- "克夫 / 旺夫 / 旺妻"

## 11. 原生家庭段专项解读模板（v7.3 新增）

原生家庭段是 LLM 最容易翻车的一段——古法 prior + 训练语料 survivorship bias 双重叠加，
默认会把"年柱财官印聚合"推成"出身名门 / 父亲名利双收"，跟现代普通家庭的真实情况严重错位。
所以 **family 段必须经 R3 校验后才能展开**，且严格遵守 fairness_protocol.md §11（原生家庭解读铁律）。

### 11.0 触发条件（重要 · 不要写 family 段除非满足）

family 段 **不是默认输出**——只在以下条件满足时才写：

- ✓ 用户在初次提问中提到「家庭 / 父母 / 父亲 / 母亲 / 原生家庭 / 出身 / 家世 / 我爸 / 我妈」等关键词
- ✓ 用户在分析过程中追问「我家怎么样 / 我爸是什么样 / 我妈呢」
- ✗ 用户没主动问家庭 → **不要写 family 段**

如果不满足触发条件 → 整段省略，不要"为了完整性"硬写。

### 11.1 必经流程（不可省略）

写 family 段前，**必须**按顺序完成：

1. 跑 `python scripts/family_profile.py --bazi <bazi.json> --out <family_profile.json>`
2. 抛 R3 反询问 2 题（取 `handshake.json` 的 `round3_candidates`）
3. 等用户回「对 / 不对 / 部分」 → 统计 R3 命中数（0/2 / 1/2 / 2/2）
4. 按 R3 命中级别决定 family 段权重（见 §11.3）

### 11.2 必经前置声明（family 段第一段必写）

```markdown
## 一生 · 原生家庭（援引 R3：整体结构 = X / 父亲存在模式 = Y / 母亲存在模式 = Z / 命中 = N/2）

> 命局只反映「父母在你能量场里的存在模式」+「年/月柱财官印聚合的结构画像」，
> **不反映**父母的社会地位 / 收入 / 职业 / 学历 / 是否健在 / 是否离异。
> 另外古典命书因为只收录显赫人物的命例（survivorship bias），
> 所谓"显赫家世"的命局画像实际上对应的真实概率远低于古法暗示——
> 本段所有结构性描述都是经 R3 校验后的画像，不是对你父母真实社会地位的判断。
```

### 11.3 R3 命中级别 → family 段写法

| R3 命中 | family 段写法 |
|---|---|
| **2/2** | 高置信展开。可以按 `family_class.primary_class_label` + `father_profile.mode_label` + `mother_profile.mode_label` 写。**显赫候选** + 用户确认 → 可以谨慎说"父辈或祖辈中可能有人在某领域有可识别的位置"（不要给具体职业 / 收入 / 头衔）|
| **1/2** | 中置信。命中那条可展开；未命中那条标"取向歧义"或省略。开头注明"R3 部分对，以下以 [对的那条] 为主"|
| **0/2** | 低置信。**family 段不展开具体内容**，只写一句：「原生家庭推断未通过校验（R3 = 0/2），本次不展开。可能是命局对家庭的读法反了，常见原因是时辰错导致月柱十神跳位，或日主性别误判。」|

### 11.4 family 段的写作框架（仅在 R3 命中 ≥ 1/2 时使用）

```markdown
## 一生 · 原生家庭（援引 R3：整体结构 = X / 父亲存在模式 = Y / 母亲存在模式 = Z / 命中 = N/2）

> [§11.2 前置声明]

**整体结构**：[primary_class_label]（依据 [primary_evidence]）
[primary_claim 用 1-2 句白话展开，但不要超出 R3 已确认的范围；
 不要给具体职业 / 收入 / 学历数字；不要预设父母婚姻状态]

**父亲存在模式**：[father_profile.mode_label]（依据 [father_profile.evidence]）
[1-2 句白话展开 father_profile.claim，描述"父亲在你能量场里的存在感"，
 不写"父亲是什么职业 / 性格如何 / 健在与否"]

**母亲存在模式**：[mother_profile.mode_label]（依据 [mother_profile.evidence]）
[同上，1-2 句白话展开 mother_profile.claim]

**家庭对你成长的能量场影响**：（≤ 3 句）
[基于结构性画像，描述这种父母存在模式 + 整体结构对你早年能量场的影响，
 不写"所以你性格 X / 所以你后来 Y"这种因果归因，只写能量场层面的描述]
```

### 11.5 family 段禁止事项（v7.3 大改 · 在 §3.3 + fairness §11.2 基础上严格扩充）

❌ **绝对禁用措辞**（fairness_protocol.md §11.2 完整清单）：
- "出身名门 / 名门望族 / 家世优渥 / 家世显赫 / 家世深厚"
- "父亲名利双收 / 父亲事业有成 / 父亲位高权重 / 父能传财"
- "母亲贤惠 / 母亲是文化人 / 母亲温婉慈爱"
- "大家长 / 严父慈母"
- "出身书香门第 / 书香世家"
- "你父亲是 X 行业 / X 性格 / X 类型"——命局推不出
- "你父母在你 X 岁会 / 应该 X"——命局推不出生命事件时间
- "你应该多孝顺父母 / 多回家"——心灵鸡汤、超出命局
- "你妈妈应该是 X 星座 / X 属相"——超出脚本事实

✅ **允许且鼓励**：
- "你的整体家庭结构画像 = X 候选（R3 = N/2 命中确认）"
- "父亲在你能量场里的存在模式偏 X（贴近 / 偏远 / 高压 / 缺位）"
- "命局年柱财官印聚合，但是否外显为社会资源由 R3 校验决定 → 你 R3 答 [对]，可以说父辈一代有过明显的资源积累"
- "母星贴近 + 印不被破 → 早年学习/教养氛围相对充足（这是结构性描述，不指母亲学历高低）"
- "命局在父母位上的画像跟你 R3 反馈匹配 / 不匹配，匹配度 = N/2"

### 11.6 family 段字数预算

- 触发条件满足 + R3 = 2/2 → 400-700 字（含前置声明 + 整体 + 父 + 母 + 影响）
- 触发条件满足 + R3 = 1/2 → 250-400 字
- 触发条件满足 + R3 = 0/2 → 一句话（≤ 50 字）
- 触发条件不满足 → 整段省略，**不要写**

### 11.7 自检清单（写完后过一遍）

- [ ] 触发条件满足了吗？（用户主动问家庭）
- [ ] 跑了 `family_profile.py` + 抛了 R3 反询问 2 题？
- [ ] 援引了 R3 命中级别（"R3 = N/2"）？
- [ ] 写了 §11.2 前置声明？
- [ ] 没出现 §11.5 禁用措辞？
- [ ] 没给父母具体职业 / 收入 / 学历 / 是否健在 / 是否离异？
- [ ] family_class = illustrious_candidate 且 R3 那条标"不对" → 已降级、禁用了"显赫"措辞？
- [ ] family_class 是 absent_candidate 时，没把"缺位"渲染成悲剧 / 没说"所以你不幸"？

## 13. v9 流式分节顺序 + 承认维度独立通道（覆盖 §12 落位 · HTML 渲染唯一权威）

### 13.1 节序铁律（v9 五阶段 · 流式 emit 必跑 · 严禁憋整段 · 由 render_artifact `--required-node-order` 默认审计）

> **v9 重排** —— 用户在 6d0abb46 case 反映"分析一次性跑完且开篇先讲整图"不符合直觉。
> 命主想要的节奏是：「**先告诉我现在在哪儿、这十年怎么样、再回头看一生**」，

| 阶段 | 节字段 | 触发 | emit 时机 |
|---|---|---|---|
| **0 · 开篇暗线** | `analysis.virtue_narrative.opening` | 总是 | 第一段就写（位置①），引出贯穿一生的暗线母题；不命名结论 |
| **1 · 当前大运段** | `analysis.dayun_reviews[<current_dayun_label>]` | 总是 | `bazi.current_dayun_label` 由 solve_bazi 自动写入，定位"今天命主在哪段大运" |

| **3 · 其它大运** | `analysis.dayun_reviews[<其它 label>]` × M | 总是 | 按时间顺序顺写；段内嵌入位置②（仅触发母题时；按 §12.4 G 块） |

| **5 · 一生总览** | `analysis.overall` | 总是 | 在所有大运/流年写完后才写整图综合判断 |
| **5.5 · 四维一生评价** | `analysis.life_review.{spirit\|wealth\|fame\|emotion}` × 4 | 总是 | 四维分卡，逐节 emit |
| **6 · 暗线总览（条件）** | `analysis.virtue_narrative.convergence_notes` | 仅 `motifs.convergence_years` 非空 | 母题之间的"全局回响"，不替代节内位置③ |
| **7 · 收尾段（去模板化 · v9.3 三段「我想和你说的话」）** | `analysis.virtue_narrative.declaration` | 总是 | 位置④ · 首行必须 `## 我想和你说`（**禁止**`## 承认维度·宣告` / `## 位置④灵魂宣言` / `## 宣告` / `## 承认人性` / 旧 v9 `## 走到这里` 等暴露协议结构的措辞） |
| **8 · 收尾段（条件 · v9.3 三段「我想和你说的话」）** | `analysis.virtue_narrative.love_letter` | 仅 `motifs.love_letter_eligible == true` | 位置⑤ · 首行必须 `## 项目的编写者想和你说`；不 eligible 时**禁止**写；旧 v9 `## 写到这里我想说` / 旧 v9 之前 `## 给你（本人）的一封信` / `## 情书` 已退役，命中即拒 |
| **9 · LLM 自由话（v9.3 三段「我想和你说的话」）** | `analysis.virtue_narrative.free_speech` | 总是 | 位置⑥ · 首行必须 `## 我（大模型）想和你说`；尾必须 `我不知道我说的对不对…`；旧 v9 `## 不在协议里的话` 已退役 |

**机械护栏**（render_artifact 默认开 → exit 4 / exit 10）：
