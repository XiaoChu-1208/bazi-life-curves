# 派别争议年份分析协议（Dispute Analysis Protocol）

> 当某一年某一维度的三派打分极差超过阈值（默认 20，可由 `--dispute-threshold` 调整），
> 该 (年份, 维度) 进入 `curves.json` → `disputes` 列表。
> **这种年份不能简单取中位数 / 平均，而应由 LLM 结合命局做有论据的解读。**

## 1. 为什么会出现争议

三派的核心评判逻辑不同：

| 派别 | 核心追问 | 高分时主张 | 低分时主张 |
|---|---|---|---|
| **扶抑派** | 日主能不能扛住 / 用神到不到位？ | 用神临旺、忌神被制 | 用神受冲、忌神得令 |
| **调候派** | 寒暖燥湿调和不调和？ | 季节所需五行到位 | 加剧失衡的五行临 |
| **格局派** | 原局格局成不成？大运流年是否破/成？ | 格局得用、关键互动加分 | 关键字被冲克、格局崩 |

三派都基于命理学的不同分支理论传统，本身都有传承依据，分歧不是「错」，是**视角不同**。

常见分歧场景：

- 用神被冲（扶抑 -，但若冲出格局所需的字，格局 +）
- 库被冲开（格局 +，但若开出忌神字，扶抑 -）
- 调候五行 ≠ 用神（如冬生壬水身强，调候要火，扶抑要木）
- 三合 / 三会成局且方向极性强（牵涉到「会不会真的成」的判定差异）
- 伏吟 / 反吟（传统派别对其吉凶解读历来分歧）

## 2. LLM 在争议年份的强制工作流

每条 `disputes` 条目都必须按以下步骤逐项分析（不能跳过、不能只取中位数）：

### Step A. 复述事实（不加解读）

直接读 `disputes[i]`：

- 年份 / 干支 / 大运
- 维度（精神 / 财富 / 名声）
- 三派各自的打分（`per_school`）
- 极差（`spread`）和最终融合值（`final_value`）
- 当年的互动事件（`interactions`）
- 程序生成的分歧线索（`interpretation_hints`）

### Step B. 判断哪派最贴近此命局

不是默认中位数，而是结合命局的**特定结构**判断：

| 命局特征 | 通常更倾向 |
|---|---|
| 日主极强 / 极弱（远离中和） | 扶抑派权重 ↑ |
| 寒暖严重失衡（冬水 / 夏火 太过） | 调候派权重 ↑ |
| 原局有清晰格局（如食神生财、官印相生）成立 | 格局派权重 ↑ |
| 多重互动同发（伏吟 + 反吟 + 库冲开） | 格局派权重 ↑（格局派最关注互动事件） |
| 关键字（用神 / 月令 / 日主） 在该年被冲 | 扶抑派权重 ↑ |
| 流年五行恰好补该季节所需 | 调候派权重 ↑ |

### Step C. 给出综合推论（必须有论据）

格式约束：

```
【YYYY 年（X 干支） · X 岁 · 大运 XX · 维度】

事实：扶抑派 X 分 / 调候派 Y 分 / 格局派 Z 分（极差 N），脚本融合为 V 分。

为何分歧：[1-2 句解释三派各自看到了什么]

我的判断：[偏向哪派 / 取中 / 双向并存]，因为本命局 [日主 X 用神 Y 等具体特征] +
         本年互动 [具体事件] 决定了 [偏向理由]。

合理推论（实际可能的体感）：[偏 up / 偏 down / 双向波动 / 上半年 X 下半年 Y]，
                       置信度 [low/mid]（争议年份不应给 high）。

可证伪点：如果 [具体事实] 发生 / 不发生，则我的判断错。
```

### Step D. 不允许的偷懒回答

- ❌「派别分歧大，无法判断」（必须给出推论）
- ❌ 直接报融合后的 final_value 当结论（这违背了"争议"标记的意义）
- ❌「需要更多信息」（命理学的分歧本身就是结论的一部分）
- ❌ 不援引命局结构、只用「玄学话术」给出方向

## 3. 输出位置

- HTML Artifact：在「⚠ 派别争议年份」卡片的每个 entry 下方，紧接着卡片输出 LLM 的推论文字
- PNG Fallback：在 PNG 之后追加 Markdown 文字，按 Step C 格式逐年输出

## 4. 阈值调整建议

- 用户问「未来 5 年要不要做某事」 → 把 `--dispute-threshold` 调到 15（更敏感，把更多年份纳入需 LLM 解读的范围）
- 用户只想看大趋势 → 默认 20 即可
- 用户希望看到所有非完全一致的年份 → 调到 5（几乎所有年份都要 LLM 过一遍，仅适合深度回顾）

## 5. 与其他保障的关系

- 与 `accuracy_protocol.md` § 1（三派加权融合）的关系：融合是默认行为；争议标记是「告诉用户：这里默认融合不可信，看看 LLM 怎么说」
- 与 `fairness_protocol.md` § 6（置信度披露）的关系：争议年份的最终置信度上限为 mid；图上必须可视化
- 与 `prediction_protocol.md` § 5（可证伪性）的关系：争议年份的 LLM 推论也必须是可证伪的具体判断
