# 预测性保障协议（Prediction Protocol）

不只描述过去，必须对未来流年给出**可证伪**的预判。

## 1. 权重模型（公式化，不是「感觉」）

```
year_value[t] = α · L0_baseline + β · L1_dayun(t) + γ · L2_liunian(t)
```

默认权重：α = 0.30, β = 0.40, γ = 0.30；约束 α + β + γ = 1。

```
cumulative[t] = Σ_{k≤t} λ^(t-k) · year_value[k] · normalization
```

衰减系数 λ：精神 0.85 / 财富 0.92 / 名声 0.96（详见 methodology.md § 4）

## 2. 拐点前瞻表（必须输出）

每次出图必须附带「未来 10 年关键拐点」表：

```
| 年份 | 干支 | 大运 | 触发条件 | 维度 | 方向 | 强度 | 置信度 |
|---|---|---|---|---|---|---|---|
| 2031 | 庚戌 | 辛卯 | 大运辛卯入墓 + 戌冲辰、伏吟年柱 | 精神 | ↓ | 中 | 高 |
| 2034 | 癸丑 | 辛卯 | 流年财库被冲开 + 食神生财 | 财富 | ↑ | 高 | 中 |
| ... |
```

**触发条件**必须是命理学具体描述（不能是「这一年看起来不太顺」）。

## 3. 置信度三档

| 置信度 | 含义 | 触发条件 |
|---|---|---|
| **high** | 三派一致 + 单一明确互动 | 极差 ≤ 10 且 `interaction_dense=False` |
| **mid** | 三派轻微分歧 或 互动稍密 | 极差 11–20 或 单一中等互动 |
| **low** | 派别分歧大 / 多重互动 / 大运过渡 | 极差 > 20 或 `interaction_dense=True` 或 大运首尾两年 |

低置信度年份在图上灰阶，文字提示「该年命理结构复杂，预测置信度较低」。

## 4. 置信带（Confidence Band）

每条线带 ±σ 阴影：

```
σ_year = base_sigma · (1 + divergence_factor) · interaction_factor
```

- `base_sigma`：默认 5（满分 100 的 ±5%）
- `divergence_factor`：三派极差 / 20
- `interaction_factor`：1.0 / 1.3 / 1.6（按互动密度）

近期年份 σ 更小，远期年份 σ 累积放大（远期不确定性更高）。

## 5. 可证伪性

每个**高置信度**预测必须具体到可被现实推翻：

- ✅ 「2028 年 Q2 之后现金流低位（-15 ~ -25 相对当前基线），原因：流年伤官见官且无财通关」
- ❌ 「2028 年财运变化」（无法证伪）
- ❌ 「2028 年要小心」（无法证伪）

## 6. 校准接口（个人化）

用户可选用过去 3–5 个关键年的实际感受（高 / 中 / 低 + 三维度）→ `scripts/calibrate.py --personal`：

- 调整 α / β / γ 权重（在 ±0.1 范围内）
- 调整 λ 衰减系数（在 ±0.05 范围内）
- 校准结果写入 `personal/<hash>.yaml`，下次该用户调用时自动加载
- **不**写回主 `calibration/dataset.yaml`（公正性要求）

## 7. 预测的时间衰减原则

预测能力随时间距离衰减：

| 时间窗 | 置信度上限 |
|---|---|
| 0–3 年 | high 可达 |
| 4–7 年 | mid 上限 |
| 8–10 年 | mid 下限 |
| > 10 年 | 仅作趋势参考，不出具体拐点表 |

`render_artifact.py` 会在 10 年后的曲线上加渐变透明度，提示远期不确定性。

## 8. 不做的预测

- 寿命具体年份
- 死亡 / 重大疾病的具体时间
- 子女数量 / 性别
- 配偶具体特征
- 任何吉凶定性结论

本 skill 只输出**结构性趋势**和**命理互动事件**，不做命定式断言。
