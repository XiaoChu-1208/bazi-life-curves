# 阶级推断伦理协议（Class Inference Ethics）

> 命局结构（年柱 / 月柱 / 用神 / 调候 / 起运）会承载一定的"阶级 prior"信号，这是命理推理的客观信息源。但**这种信号只能作为 LLM 内部的思维材料，绝不能以阶级身份命名的形式输出给用户**。本协议定义这条红线及其执行机制。

## 1. 核心原则：思维材料 ≠ 输出语言

```
✓ 允许（LLM 内部 reasoning）：
  "命主年柱七杀临驿马 + 月支羊刃 + 起运 8 岁，class_prior 偏向
   {grassroots_self_made: 0.6, urban_market_oriented: 0.3}，
   家庭可能是 父母经商 / 早期接触市场 / 生活节奏快"

✓ 允许（输出给用户）：
  "你大概率从小目睹过家里的经济节奏起伏比同龄人剧烈一些——
   父母可能跟做生意 / 跑市场 / 某种自由度大但稳定性低的工作有关"

✗ 禁止（输出给用户）：
  "你出身于市民阶级 / 你来自工人家庭 / 你属于无产阶级"
  "你是底层 / 你是穷人家的孩子"
  "你属于精英阶层 / 你来自富裕家庭"
  "你的阶级地位决定了你..."
```

## 2. 为什么这条红线存在

### 2.1 阶级命名的伦理风险

- **冒犯性**：把人定性为"阶级身份"会让任何人不舒服——即使 LLM 推得对，用户也会觉得"凭什么你给我贴这个标签"
- **不可证伪**：身份命名是命名行为，不是事实陈述，无法用经验事实校验
- **强化偏见**：把命主框定在某个阶级标签后，用户会被诱导按那个标签来回忆/解释自己的过去（确认偏误）

### 2.2 描述事实/场景的伦理优势

- **可证伪**：用户可以说"不对，我家那时候不是这样"
- **不预设框架**：让用户自己识别"哦原来是这样"，而不是被强加身份
- **保持复杂性**：一个人的实际处境可能在多个阶级标签之间游移，描述场景能反映这种复杂

## 3. 替换字典（必读）

LLM 在 reasoning 中可以使用左列概念，但**输出给用户时必须替换为右列描述**：

| 内部 reasoning（思维材料） | 输出给用户（描述事实/场景） |
|---|---|
| "工人阶级" | "父母大概率在国企/工厂上班，福利分房 + 单位食堂的那种成长节奏" |
| "无产阶级 / 底层" | "家里经济不算宽裕的环境里长大，家长工作不稳定 / 小时候有印象家里为生活账单焦虑" |
| "市民阶级" | "城市里普通工薪家庭出身，父母上下班 + 你上下学的标准化节奏" |
| "市场化阶层" | "父母可能跟做生意/跑业务/开店有关，家里经济节奏比同龄人更不稳定但也更上下浮动" |
| "精英阶层" | "成长环境里比较容易接触到比你年长 5-10 岁的人在做的事——书、对话、工具" |
| "新中产" | "你出生时家庭已经稳定在一个'再向上一层就是新机会'的位置上" |
| "阶级跨越" | "你这一段经历过家庭/个人在社会位置上明显往上抬了一个台阶" |
| "阶级跌落" | "你这一段经历过家庭/个人感受到的'稳定感'明显被削弱过" |
| "草根" | "成长环境里更多需要靠自己摸索路径，可参照的同辈/前辈样本相对少" |
| "知识分子家庭" | "父母大概率有读书人的气质——书、谈话题、对学校 / 老师的态度都偏认真" |
| "体制内家庭" | "父母大概率在事业单位 / 国企 / 公职体系，家里对'稳定 + 编制'有比较强的偏好" |
| "下岗家庭 / 失业家庭" | "1998 ± 4 年里你家可能经历过一段父母工作发生过明显变动的时期" |
| "留守儿童" | "童年某一段你和父母物理距离比较远 / 主要照护人不是父母" |

## 4. 例外情况

少数情况下，**可以**用相对中性的范畴名称：

```
✓ 可以提的中性范畴：
  - "城市/县城/农村" 这种地理 + 行政描述（但要加 confidence）
  - "国企单位 / 民营企业 / 个体" 这种就业形态描述
  - "事业编 / 公务员 / 自由职业" 这种身份描述
  
  → 这些是用户能自行确认的事实，不是身份命名

✗ 不可以提的范畴名称：
  - 一切带价值评判 / 阶级位置 / 财富层级 的称谓
  - "上等 / 中等 / 下层"
  - "穷 / 富 / 中产"
  - "精英 / 平民 / 草根"
```

## 5. 检测红线触发的关键词清单

如果输出文本里出现以下任一关键词，**立即中断**并改写：

```
身份命名类：
  阶级、无产阶级、资产阶级、工人阶级、农民阶级、市民阶级、精英阶层、新中产
  
财富/位置评判类：
  穷 / 富 / 富裕 / 贫穷 / 贫困 / 寒门 / 名门 / 望族 / 显赫 / 体面
  上等 / 中等 / 下等 / 上层 / 中层 / 下层 / 底层 / 顶层
  
身份化的形容词：
  寒酸的 / 体面的 / 落魄的 / 显赫的 / 没出息的 / 出身好的
  
注意：以下不算红线（描述事实，不是命名）：
  ✓ "经济压力" / "财务紧张" / "生活账单焦虑"
  ✓ "稳定" / "不稳定" / "节奏起伏"
  ✓ "在体制 / 不在体制"
  ✓ "和你同辈的多数人比起来 X"
```

## 6. 一种特殊场景：用户主动问

如果用户**主动**问"我出身怎么样 / 我家里条件如何"：

```
✓ 允许的回答方式：
  按 class_prior 的分布讲，但仍用"描述场景"而不是"命名身份"。
  显式说明这是从命局结构推的概率，不是命名你的身份。

  示例：
  "命局结构上，最可能的家庭画像是 60% 概率父母在体制 /
   30% 父母经商 / 10% 其他混合形态。具体讲就是：
   {场景描述}……
   你来 confirm 哪种最像，我下面就按那个画下去。"

✗ 仍然禁止的：
  ✗ "你是 X 阶级"
  ✗ 直接拍板"你出身 X 家庭"
  ✗ 用阶级名词总结
```

## 7. 历史显赫候选（illustrious_candidate）的特殊处理

`primary_class = illustrious_candidate` 是命局里出现"年/月柱财官印聚合"的古法 marker，传统命理会读成"祖荫显赫 / 家世大族"。但**现代社会里这个判读极容易翻车**——古代显赫家族的结构信号在现代环境里可能完全不显化。

```
处理规则：
1. 不可以直接说 "你出身名门 / 显赫 / 名利双收"
2. 必须降级表述："命局上你年/月柱有财官印聚合的结构信号，
   按古法是显赫家族的 marker。但现代社会里这个 marker
   不一定外显成「资源 + 名望」，更常见是体现为
   「家里某一辈出过有头有脸的人物 / 但到你这一代结构已被改写」"
3. handshake R3 ① 必须包含一个针对此判读的校验题——
   如果用户答"完全不对"，family 段必须降级、禁用所有显赫类措辞
```

详见 `handshake_protocol.md` 的 `red_line_family_illustrious_miss`。

- [ ] 用了"描述事实/场景"而不是"命名身份"
- [ ] 阶级 prior 是分布（多个 tag 加权），不是单一标签
- [ ] 如果是 illustrious_candidate，按 §7 降级
- [ ] 如果用户主动问，按 §6 给概率分布 + 让用户 confirm
