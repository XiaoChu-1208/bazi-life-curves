# 怎么用（用户视角）

这份文档面向**最终用户**——你只需在 Claude / Cursor 对话中说一句话，剩下的 skill 自动跑。
开发者请看 `SKILL.md`、安装请看 `INSTALL.md`。

---

## 30 秒速览（单盘）

```
你：画一下八字 庚午 辛巳 壬子 丁未 男 1990 的人生曲线

Claude：
  ① 跑脚本算命局 + 三派打分 + 盲派事件 + 感情独立通道 + 拐点  （5-10 秒）
  ② 【v6 新增】先抛 R0 反询问·感情画像 2 题 ——"取向校准"   ← ⚠ 必须等你回应
     ① 偏好类型（你历史上喜欢过的人，是不是属于"XX 型"那一挂？）
     ② 对方态度（你被吸引的那段关系里，对方对你是哪种态度？）
     · 用于校准我该怎么读你这盘（看格局还是扶抑），以及辅助判断时辰
  ③ 再抛 R1 健康三问 ——"命局校准"
     ① 寒热出汗（你从小怕热还是怕冷？出汗多吗？）
     ② 睡眠精力（睡得浅还是沉？精力起伏大不大？）
     ③ 易病脏腑（哪个脏腑系统老出问题？）
  你回每题"对 / 不对 / 部分"
     · 双层判定（v6）：取向准确度（R0 命中）+ 命局准确度（R1 命中）
     · R1 = 3/3 → 直接进入下一步
     · R1 = 2/3 → 我再追 3 条"本性 + 过往"问题（合计 ≥4/6 才放行）
     · R1 ≤ 1/3 → 八字不准，请核对时辰
     · 红线：感情①（偏好类型）✗ + R1 ≤ 1/3 → 立即停手，要时辰
  ④ 校验通过后**问你一句**：要 markdown 流式（A · 默认）还是 HTML 交互图（B · 多等 5-15 秒）
  ⑤ 在对话里**流式输出**整图综合分析 → 一生四维度（精神/财富/名声/感情）→ 大运每段（含感情看点）→ 关键年份每条
     · 每写完一节立刻发出，你能边读边等下一节，不用等所有内容写完
  ⑥ 仅当你选 B：把流式内容打包成交互式 HTML Artifact（**8 条曲线** · 可展开 / 收起 details 段）
```

> 想省时间？回 A 就行。想要图、想后续保存 / 转发整页 → 回 B。

## 30 秒速览（合盘）

```
你：我的八字是 X，我老公的是 Y，我们婚配怎么样？

Claude：
  ① 分别解析你和你老公的八字
  ② R0 反询问·感情画像 2 题 —— **只问你自己的**（v6）
     · 老公那边的偏好/态度从你这一盘的「夫宫 + 财星」直接读出，不再问你
  ③ 分别给你和你老公的 R1 健康三问
     · 你自己的：直接回答
     · 你老公的：你能答多少答多少（答不上的会标 caveat）
  ④ 跑合盘评分（4 层：五行互补 + 干支互动 + 十神互配 + 大运同步）
  ⑤ 在对话里**流式输出**（每写完一节立刻发，不用等全部）：
     · 概览 + confidence
     · 4 层评分逐层解读
     · 关键加分 / 减分项 → 怎么用 / 怎么避
     · 未来 5-10 年大运同步度 → 哪些年适合一起搞大事
     · 关系类型 tips（婚配 / 合作 / 友谊 / 家人 各有侧重）
```

> 合盘默认就是 markdown 流式，不渲染 HTML（合盘是关系解读，没有"曲线图"那种刚需图的内容）。想要 HTML 汇总表请明说"也给我个汇总表 / HTML"。

如果你在 Step ② 答 "都不对"，Claude 会**拒绝**继续，让你先核对八字是否准确（多见于时辰偏 1 小时）。

## 触发短语（任选一种）

### A. 单盘 - 已经知道四柱
```
画一下八字 庚午 辛巳 壬子 丁未 男 1990 的人生曲线
画八字曲线 戊寅 甲辰 戊寅 庚申 男 1977
我的八字：丁巳 甲辰 戊寅 庚申，1977 年男，看看大运走势
```

### B. 单盘 - 只有公历生辰
```
我的公历生辰是 1990-05-12 14:30 男，看看 50 年大运曲线
1982-01-15 早上 9 点出生，女，画人生曲线图
```

### C. 单盘 - 含定制参数
```
画 30-55 岁这一段的曲线就行                 → --age-start 30 --age-end 55
看看未来 5 年的拐点                          → --forecast-window 5
分歧大的年份多标一些（敏感模式）              → --dispute-threshold 15
只看大趋势，别太敏感                         → --dispute-threshold 25
```

### D. 合盘 - 婚配 / 夫妻
```
我和我老公合盘：我 庚午 辛巳 壬子 丁未 男 1990，他 甲子 丙寅 戊午 庚申 女 1984
看看我和我对象配不配，我八字是 X 男 1992，对象 Y 女 1993
我们 1990-05-12 14:30 男 + 1992-08-23 09:15 女，做个夫妻合婚
```

### E. 合盘 - 合作 / 创业
```
我和合伙人合盘看合作怎么样：我 X 男 1985，他 Y 男 1988
团队三人合盘合作：CEO X / CTO Y / COO Z（各自八字）
未来 5 年我们仨适合一起干吗
```

### F. 合盘 - 友谊 / 家人
```
我和这个朋友合盘看友谊：X / Y
我和我妈八字合一下，看亲子关系：我 X，妈妈 Y
```

参数 + 关系类型会被 Claude 自动识别，不需要你手写命令。
若关系类型没明说，Claude 会先问你（合作 / 婚配 / 友谊 / 家人 4 选 1）。

## 你会拿到什么

### 0. 流式输出 + 输出格式选择（v5 新增 · 默认更快）

校验通过之后 Claude **会主动问你一句**：

```
(A) 纯 markdown 流式输出 —— 我每写完一节就立刻发给你，最快、最适合手机 / 复制 / 转发
(B) markdown 流式 + 最后渲染 HTML 交互图 —— 多等 5-15 秒
```

| 你想要的 | 怎么说 / 怎么回 |
|---|---|
| **要最快、文字够用**（默认） | 回 `A`，或干脆什么都不说（默认 A） |
| **要可以悬停看每年的图、要保存整页 HTML** | 回 `B`，或在第一句就说"画图 / 出 artifact / 给我图" |
| **不想被问，要图** | 第一句就说 "画八字曲线 + HTML / 给我交互图" |
| **不想被问，只要文字** | 第一句就说 "口头给我说说就行 / 不用图" |

合盘默认 markdown only —— 合盘没有"曲线图"那种刚需图。

无论 A 还是 B，**所有文字分析都是流式输出**：每写完一节立刻发给你，你能边读边等下一节，不用等所有内容写完。

## 为什么需要 Step ② 校验？v6 改成"双层校验"了

八字最敏感的是**时辰**——出生时辰差 1 小时，月柱 / 时柱可能跳一位，
整个曲线 / 合盘就完全不同了。**而同样的八字"该走格局还是该走扶抑"** 也常常需要外部信息校准。

所以 v6 起 skill 会先抛**两轮**问题：

### Round 0：反询问·感情画像（v6 新增 · 取向校准 · 2 题）

由系统**主动**抛给你两道题，把你历史上的感情侧面画出来——
这不是为了"算桃花"，而是用来**校准我该用哪派方法读你这盘**：

1. **偏好类型**（你被吸引的人，更接近哪种特质？给出 3-4 个选项，每个选项都是从你八字的"妻/夫星五行 + 夫宫主气十神"翻译过来的人话）
2. **对方态度**（你被吸引的那段关系里，对方对你是更主动 / 更被动 / 更依赖 / 更挑剔？同样从你八字的「妻/夫星强弱 + 比劫 + 食伤 + 印 + 桃花」翻译过来）

> 为什么先问感情：感情画像在八字里的"指针"非常硬（妻/夫星几乎一对一翻译成一类人），
> 你对它的回答 = 一次"取向 + 时辰"双重投票。

### Round 1：健康三问（命局校准 · 3 题）

1. **寒热出汗**（你从小怕热 / 怕冷？出汗多不多？）
2. **睡眠精力**（睡得浅 / 沉？精力起伏大不大？）
3. **易病脏腑**（哪个脏腑系统老出问题？）

为什么用健康问题而不是过往大事：

- **健康/体感记忆终生稳定**：你对"从小怕不怕冷"的回答几乎是二值的，记不糊涂
- **错八字一查就露馅**：时辰错 1 小时 → 命局 climate 跳档 + 五行权重剧变 → 三条全跳，不可能蒙混
- **三个不同侧面 = 三次独立投票**：单条偶然撞中的概率几乎为零

### v6 双层准确度判定

| 取向准确度（R0 命中） | 命局准确度（R1 命中） | 整体结论 | Claude 行动 |
|---|---|---|---|
| 2/2 | 3/3 | **高** | 全速放行，按格局派优先解读 |
| 2/2 | 2/3 | **中-高** | 进入 + R0 高置信 → 感情段重点写；R1 标 caveat |
| 1/2 | 3/3 | **中** | 进入；感情段双解（两种偏好都讲一点） |
| 1/2 | 2/3 | **中** | 追 R2 三条交叉，≥ 4/6 才进；感情段双解 |
| 0/2 | 3/3 | **中-低** | 进入；感情段**必须双解**，主调改"扶抑/调候" |
| 任意 | 1/3 | **低** | 同上 + 强 caveat；R2 仍不达 4/6 → 拒绝 |
| 任意 | 0/3 | **拒绝** | 不再追问，强烈建议复核八字（多半时辰错） |

**红线**（命中其一立即停手要时辰）：
- 寒热出汗 ✗ 或 易病脏腑 ✗ → 命局根盘判错
- **★ v6 新增**：感情①·偏好类型 ✗ + R1 ≤ 1/3 → 妻/夫星 + 健康同时崩盘，几乎可断定时辰偏 ≥ 1 小时

这是**硬门槛**——避免在错八字上做大量徒劳分析。

## 关于"LLM 推论"的几个常见疑问

### Q1：为什么有些年份的"LLM 推论"显示占位文字而不是真正的分析？

A：LLM 推论是 Claude **在对话里现写**的，理论上每个争议年份 + 每个拐点都应该写。
如果你看到占位符（"此处由 LLM 写入..."），说明：
- **打开的是示例 HTML**：示例只填了少数几个年份做演示（比如只填了 1990、2027、2032、2036）
- **Claude 在 Step 3a 偷懒了**：可以直接告诉它"把所有 disputes 和 turning_points 都写满"

### Q2：LLM 推论的内容靠谱吗？

A：靠 3 个机制保证：

1. **数字必须来自脚本**：LLM 不能自己给分数，只能解释脚本输出
2. **必须援引盲派应事 + 命局结构**：禁止"需要综合判断 / 因人而异"这种空话
3. **必须写可证伪点**：比如"如果该年并未发生 X，这一条就是错的"——便于你自己验证

详细规则见 `references/multi_dim_xiangshu_protocol.md` + `references/dispute_analysis_protocol.md`。

### Q3：为什么不能 LLM 直接打分？

A：因为这正是命理界的老问题——同一八字，不同师傅给出完全相反的判断。
本 skill 的设计哲学是：

- **"打分 = 算法 + 历史回测"**：保证公正、可重复、可证伪
- **"解读 = LLM + 强制协议"**：保证有论据、可质疑、能落到具体行动

把这两件事拆开，既不放弃命理的丰富性，也不让结果完全依赖某个 AI 的"灵感"。

### Q4：盲派为什么不进 25% 融合权重？

A：因为盲派擅长"应事 / 应期"（具体哪一年发生什么），
但**不擅长给整体能量打分**——盲派的师承本来也不做这件事。

所以 skill 把盲派定位为：
- **事件断**：识别 11 类经典组合（伤官见官 / 比劫夺财 / 财库被冲开 ...）
- **烈度修正**：在三派融合分上 ±8（重）/ ±4（中）/ ±2（轻）

详见 `references/mangpai_protocol.md`。
